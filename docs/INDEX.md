## ğŸ”„ **Documentation Maintenance**

### **How to Keep Docs Updated**

**Use BMAD Workflows:**

1. **`@bmad/bmb/workflows/redoc`** - Autonomous documentation maintenance

   - Maintains module and workflow docs using reverse-tree approach
   - Use when: Major documentation updates needed

2. **`@bmad/bmm/workflows/document-project`** - Brownfield documentation
   - Analyzes codebase and creates comprehensive docs
   - Use when: Need to document existing code

### **Manual Update Guidelines**

**After completing a story:**

- [ ] Update [`sprint-status.yaml`](sprint-status.yaml) status
- [ ] Create story summary in [`stories/`](stories/)
- [ ] Update [`../.cursor/.cursor-changes`](../.cursor/.cursor-changes)
- [ ] Update [`README.md`](../README.md) if new features added

**After completing an epic:**

- [ ] Run retrospective: `@bmad/bmm/workflows/retrospective`
- [ ] Update architecture if decisions changed
- [ ] Archive outdated documents to [`archive/`](archive/)
